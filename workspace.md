# StellarParameterHybridNet — Project Workspace

## Overview

A PyTorch pipeline for predicting fundamental stellar atmospheric parameters
from optical and near-infrared spectra:

- **T_eff** (effective temperature, K)
- **log g** (surface gravity, dex)
- **[Fe/H]** (metallicity, dex)

The architecture, `StellarParameterHybridNet`, fuses two parallel streams:

1. **CNN branch** — 1D ResNet applied to continuum-normalised spectral flux.
2. **Dense branch** — fully-connected network applied to hand-crafted physical
   line features (equivalent width, FWHM, depth) extracted via Gaussian profile
   fitting on key absorption lines.

Fusion is performed by concatenation followed by a `CrossModalAttention` gate.

All `.npy` processed files and `label_stats.npy` / `feature_stats.npy` must

## Known Design Notes

- **Split-then-normalise**: feature and label z-score statistics are fit on the
  training partition only and reused for the validation partition, preventing
  data leakage.
- **MaStar per-visit duplication**: `goodspec` contains multiple visits per star.
  The same VAC label is assigned to all visits; stars from the same physical
  object may appear in both train and val partitions under different visit rows.
  Using `combspec`-only (one combined spectrum per star) eliminates this risk.
- **Cross-domain evaluation**: each survey's `evaluate_mastar.py` maps MaStar
  optical flux onto that survey's arm grid and runs inference with the
  survey-specific model. Arms outside the MaStar wavelength range are
  zero-filled; the feature branch carries the principal information signal in
  this regime.
